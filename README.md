# Document Intelligence System

Upload PDF documents, ask questions in natural language, and receive grounded answers with citations. The system runs a retrieval-augmented generation (RAG) pipeline: documents are parsed, chunked, embedded, and stored in a vector database, then relevant passages are retrieved to ground the language model's answer.

## Features

- **Multi-file PDF upload** (1–50 files) with validation and duplicate detection
- **Ingestion pipeline** — extract → chunk → embed → store, with graceful per-document error handling
- **Semantic retrieval** over a persistent vector store
- **Grounded answers** generated by an LLM constrained to the retrieved context
- **Citations** linking each answer back to its source document and page
- **Next.js frontend** wired to the backend through a same-origin proxy

## Architecture

```
Ingestion:  Upload PDFs → Extract text → Chunk → Embed → Store in vector DB
Query:      Question → Embed → Retrieve top-k chunks → Build prompt → LLM → Answer + Citations
```

Each stage is an isolated service; a thin orchestration layer sequences them, and the API routes stay thin by delegating to the service layer.

## Tech Stack

| Layer        | Technology                                              |
| ------------ | ------------------------------------------------------- |
| API          | FastAPI                                                 |
| PDF parsing  | pypdf                                                   |
| Embeddings   | `sentence-transformers/all-MiniLM-L6-v2` (Transformers) |
| Vector store | ChromaDB (persistent)                                   |
| LLM          | Groq — `llama-3.3-70b-versatile`                        |
| Frontend     | Next.js, React, Tailwind CSS                            |

## Prerequisites

- Python 3.10+
- Node.js 18+
- A [Groq API key](https://console.groq.com)

## Setup

### Backend

```bash
cd backend
pip install -r requirements.txt

# configure secrets
cp .env.example .env        # then set GROQ_API_KEY in .env

uvicorn app.main:app --reload
```

The API is served at `http://localhost:8000`.

### Frontend

```bash
cd frontend
npm install
npm run dev
```

The app is served at `http://localhost:3000` and proxies API calls to the backend.

## Environment Variables

Configured in `backend/.env` (never committed):

| Variable               | Required | Default                                   | Description                     |
| ---------------------- | -------- | ----------------------------------------- | ------------------------------- |
| `GROQ_API_KEY`         | Yes      | —                                         | Groq API key for answer generation |
| `LLM_MODEL`            | No       | `llama-3.3-70b-versatile`                 | Groq model name                 |
| `EMBEDDING_MODEL_NAME` | No       | `sentence-transformers/all-MiniLM-L6-v2`  | Embedding model                 |

## API Reference

| Method | Endpoint  | Description                                          |
| ------ | --------- | --------------------------------------------------- |
| `GET`  | `/`       | API name, version, and description                  |
| `GET`  | `/health` | Health check                                        |
| `POST` | `/upload` | Upload PDFs (multipart); returns ingestion summary  |
| `POST` | `/chat`   | Ask a question; returns an answer with citations    |

**Example — ask a question:**

```bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"question": "What is the vacation policy?"}'
```

```json
{
  "answer": "Full-time employees receive 20 days of paid vacation per year.",
  "citations": [
    { "chunk_id": "handbook.pdf-p2-0", "filename": "handbook.pdf", "page_number": 2, "snippet": "..." }
  ]
}
```

## Notes

- Uploaded files and the vector database persist under `storage/` across restarts.
- If no relevant documents are found, `/chat` returns a clear message instead of a fabricated answer.
- Missing credentials or upstream LLM failures return a `503` with a descriptive message.
